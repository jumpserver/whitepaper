---
title: 4.2 架构设计
---

本节梳理 JumpServer 的核心设计：统一身份、授权模型、会话管控与审计存储，面向高可用与可扩展场景。

<img src="/whitepaper/image031.png" alt="架构设计示意" width="800" />

## 4.2.1 身份与权限模型

- 统一身份源：支持本地用户与 LDAP/AD 对接；同步组织/组信息
- 强认证：密码 + OTP/硬件令牌；基于来源网段/时段/设备的策略
- 授权模型：RBAC/ABAC 结合；临时提权与审批流；最小权限实践

## 4.2.2 会话代理与命令采集

- 协议代理：对 SSH/RDP/数据库/远程应用等协议进行代理与隔离
- 命令采集：命令与会话元数据结构化记录，支持实时与离线检索
- 风险控制：命令黑白名单、双人会话监督、水印与溯源标记

## 4.2.3 审计存储与归档

- 多介质存储：录屏与命令日志分层存储；冷热分层与压缩
- 留存策略：按合规要求设置留存周期与归档目标（对象存储/NAS）
- 可回放：高保真回放，支持导出与证据保全

## 4.2.4 可扩展与可观测

- 解耦与横向扩展：控制面/数据面分层，无状态组件易扩容
- 指标与日志：关键路径埋点与统一日志汇聚，支持追踪与告警
- 可集成：开放 API 与 Webhook 便于接入工单、CMDB、CI/CD
